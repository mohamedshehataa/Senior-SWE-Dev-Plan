# Daily Concepts Learning

## Goal
Learn and master one concept deeply every day (alternating between domains)

## Schedule
- **Monday/Wednesday/Friday**: System Design concepts
- **Tuesday/Thursday**: Java/Spring/Distributed Systems concepts

## Structure
Each concept includes:
1. Theoretical explanation
2. Practical code examples
3. Real-world use cases
4. Interview questions
5. Hands-on exercise

---

## Concept Categories

### 1. Java & Spring Framework
- Core Java (Concurrency, Collections, JVM)
- Spring Boot (IoC, DI, AOP)
- Spring Cloud (Microservices)
- Spring Data JPA
- Spring Security
- REST API design
- Testing (JUnit, Mockito)

### 2. Distributed Systems
- Message Queues (RabbitMQ, Kafka, ActiveMQ)
- Caching (Redis, Memcached)
- Service Discovery (Eureka, Consul)
- API Gateway (Spring Cloud Gateway, Zuul)
- Circuit Breaker (Resilience4j)
- Distributed Tracing
- Event Sourcing & CQRS

### 3. System Design Fundamentals
- Load Balancing
- Database Sharding
- Replication
- CAP Theorem
- Consistency patterns
- Availability patterns
- Performance optimization
